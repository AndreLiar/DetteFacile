## Architecture Decision Records (ADRs)

### ADR-001 — Framework: Next.js App Router
**Decision:** Use Next.js 16 with App Router.
**Rationale:** SSG/SSR flexibility, excellent PWA ecosystem, Vercel deployment, strong community.
**Consequences:** All pages under `app/`, server components by default, client components opt-in with `"use client"`.

### ADR-002 — PDF Generation: @react-pdf/renderer (client-side)
**Decision:** Use `@react-pdf/renderer` for client-side PDF generation.
**Rationale:**
- No backend required for MVP
- React-native API (JSX-based PDF layout)
- Generates valid PDF/A files
- Works in browser via web worker
**Alternatives rejected:**
- `jsPDF` → low-level, harder to template
- `puppeteer` → requires Node.js server, overkill for MVP
- `pdfmake` → less React-idiomatic

### ADR-003 — PWA: next-pwa
**Decision:** Use `next-pwa` with Workbox for service worker generation.
**Rationale:** Zero-config PWA support for Next.js, offline caching, installable.
**Config:** Cache all static assets + runtime PDF generation logic.

### ADR-004 — Styling: Tailwind CSS + shadcn/ui
**Decision:** Tailwind CSS with shadcn/ui component primitives.
**Rationale:** Rapid UI development, mobile-first, accessible components, no runtime CSS-in-JS overhead.

### ADR-005 — State Management: React Hook Form + Zod
**Decision:** Use `react-hook-form` for form state + `zod` for schema validation.
**Rationale:** Best-in-class form handling, minimal re-renders, type-safe validation schema shared between form and PDF data.

### ADR-006 — No Backend for MVP
**Decision:** Zero backend, fully static + client-side.
**Rationale:** Fastest path to MVP. User data never leaves the browser (privacy by design). PDF generated and downloaded locally.
**Future:** Add Supabase or Prisma when auth/storage needed.

### ADR-007 — Deployment: Vercel
**Decision:** Deploy on Vercel.
**Rationale:** Native Next.js support, instant deploys, free tier sufficient for MVP.

## Data Model

```typescript
// lib/schema.ts
interface DebtFormData {
  // Prêteur
  preteur: {
    nom: string
    prenom: string
    adresse: string
    ville: string
    codePostal: string
  }
  // Emprunteur
  emprunteur: {
    nom: string
    prenom: string
    adresse: string
    ville: string
    codePostal: string
  }
  // Prêt
  pret: {
    montant: number                      // en euros
    dateRemise: string                   // ISO date
    dateRemboursement: string            // ISO date
    modalite: 'unique' | 'echelonne'
    nbEcheances?: number                 // si echelonne
    lieuSignature: string
  }
}
```
